# 既存セクション修正ガイド

テーマオリジナルセクションを安全に修正するためのリファレンス。

## 判定フロー

```
対象セクションは c- プレフィックス付き？
├── YES → カスタムセクション: テーマ規約に従って自由に編集
└── NO  → テーマオリジナル: 以下の判定フローへ
          │
          ├── CSSのみの変更？
          │   └── YES → custom.css に追記（セクションファイル不変）
          │
          ├── HTML構造の変更が必要？
          │   ├── 軽微 → セクションファイルを最小限に修正（コメント付き）
          │   └── 大幅 → c- オーバーライドセクションの新規作成を提案
          │
          └── Schema設定の追加・変更？
              └── セクションファイルを修正（コメント付き）
```

## テーマオリジナル修正時のルール

### 1. 変更箇所にコメントを残す

```liquid
{%- comment -%} CUSTOM: [変更内容と理由] - [日付] {%- endcomment -%}
<div class="custom-added-element">
  <!-- 追加したコンテンツ -->
</div>
{%- comment -%} END CUSTOM {%- endcomment -%}
```

### 2. custom.css への追記パターン

```css
/* ============================
   c-[目的]: [変更内容の説明]
   Date: YYYY-MM-DD
   ============================ */

/* セクション固有のスタイル */
.shopify-section--[section-class] .target-element {
  /* スタイル */
}

/* レスポンシブ */
@media screen and (min-width: 700px) {
  .shopify-section--[section-class] .target-element {
    /* タブレット以上 */
  }
}
```

### 3. CSS詳細度の管理

テーマオリジナルのスタイルを上書きする場合:
- `!important` は**最後の手段**。まず詳細度を上げる
- セクションクラス（`.shopify-section--*`）をセレクタに含めて詳細度を確保
- custom.css は theme.css の後に読み込まれるので、同じ詳細度なら custom.css が勝つ

### 4. 非破壊的な変更を優先

| やりたいこと | 推奨アプローチ |
|------------|-------------|
| 要素を非表示にしたい | `display: none` を custom.css に追記 |
| 色・フォントを変更したい | CSS変数の上書き（custom.css or テーマ設定） |
| 要素を追加したい | セクションファイルを最小限に修正 |
| レイアウトを変更したい | CSS Grid/Flexbox の変更を custom.css に追記 |
| 機能を大幅変更したい | c- オーバーライドセクションを新規作成 |

## c- オーバーライドセクション

テーマオリジナルの大幅修正が必要な場合、元のセクションを参考に `c-` バージョンを新規作成:

1. 元のセクションを読み込んで構造を理解
2. `sections/c-[original-name].liquid` として新規作成
3. 必要な変更を加える
4. テーマエディタで元のセクションと差し替え

**利点**: テーマアップデート時に元のセクションが上書きされても影響なし
**注意**: テーマの新機能が自動で反映されなくなる

## 修正後チェックリスト

- [ ] 変更箇所に `CUSTOM` コメントが付いている
- [ ] custom.css への追記にはヘッダーコメントが付いている
- [ ] 変更禁止ファイルに触れていない
- [ ] `!important` を使っていない（やむを得ない場合は理由をコメント）
- [ ] `/shopify-schema-validator` で検証済み
- [ ] テーマエディタでプレビュー確認が可能な状態
