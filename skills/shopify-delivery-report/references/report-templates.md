# 報告文テンプレート集

納品・進捗報告文の型。どちらの形式でも骨格は同じ 5 部構成（①対応内容サマリ ②変更点一覧 ③検証済み事項 ④確認してほしい点 ⑤次のアクション・保留事項）。

---

## Slack 向けテンプレート

簡潔・箇条書き・カジュアル敬語。挨拶は 1 行、絵文字は控えめに 0〜2 個まで。

### 構造

```
お疲れ様です！{案件・機能名}の対応が完了しましたのでご報告です。

■ 対応内容
（サマリ 1〜3 行 or 箇条書き）

■ 確認済みの動作
・（何をどう確認したか）

■ ご確認いただきたい点
・（相手のアクションが必要な事項。無ければセクションごと省略）

■ 次の予定
・（次のアクション・保留事項。無ければ省略）

{プレビュー URL（共有可と確認できた場合のみ）}
気になる点があればお知らせください！
```

### 実例

```
お疲れ様です！トップページ改修の対応が完了しましたのでご報告です。

■ 対応内容
・トップページのメインビジュアルセクションを新設（画像・キャッチコピーは管理画面から変更できます）
・スマホ表示で商品一覧が横にはみ出す崩れを修正

■ 確認済みの動作
・PC / スマホ両方で表示確認済み
・既存の他ページへの影響がないことを確認済み

■ ご確認いただきたい点
・メインビジュアルは仮画像を入れています。本番用の画像素材をご共有いただけますか？（推奨サイズ: 横 2000px 以上）

プレビューはこちらです → {URL}
気になる点があればお知らせください！
```

## 機能単位への翻訳（内部用語 → クライアント語彙）

変更点はファイル名・内部用語のままではクライアントに伝わらない。必ず「その変更で何ができるようになったか」に言い換える。

| 内部表現（そのまま書かない） | クライアント語彙（こう書く） |
|---|---|
| `sections/c-hero.liquid` 追加 | トップページのメインビジュアルセクションを新設 |
| `c-product-grid` の Layer 3 CSS 修正 | 商品一覧の表示崩れ（スマホ）を修正 |
| schema に `heading_size` setting 追加 | 見出しの文字サイズを管理画面から変更できるようにしました |
| metafield `custom.release_date` を参照 | 商品ごとに設定した発売日を表示する仕組みを追加 |
| `snippets/price.liquid` リファクタ | 価格表示の内部整理（見た目の変化はありません） |
| JS を defer 化・画像 lazyload 追加 | ページの表示速度を改善しました |

翻訳の指針:

- 「どのファイルを触ったか」ではなく「サイトのどこが・どう変わったか」を書く
- クライアントが自分で操作できるようになった点（管理画面から変更可能など）は明記する
- 見た目に変化のない内部整理は「見た目の変化はありません」と一言添える

## 悪い報告例と良い報告例

同じ作業内容でも、書き方でクライアントの理解度と安心感が変わる。

### 悪い例

```
お疲れ様です。c-hero.liquid と c-feature-list.liquid を追加し、
theme.css に Layer 3 のスタイルを約 120 行追記しました。
schema の blocks は max 6 で切ってあります。push 済みなので確認お願いします。
```

問題点: ファイル名・内部用語（Layer 3 / schema / blocks / push）のままでクライアントに意味が通じない。何を確認すればいいのか、どこを見ればいいのかも分からない。検証したかどうかも不明。

### 良い例

```
お疲れ様です！トップページの改修が完了しましたのでご報告です。

■ 対応内容
・「メインビジュアル」と「特長紹介」の 2 セクションをトップページに追加しました
・どちらも管理画面（テーマカスタマイズ）から文言・画像を変更できます
・特長紹介は最大 6 項目まで登録可能です

■ 確認済みの動作
・PC / スマホ両方で表示確認済みです

■ ご確認いただきたい点
・プレビュー {URL} からデザインをご確認いただき、問題なければ公開に進めます

気になる点があればお知らせください！
```

改善点: 機能単位で「何ができるようになったか」を説明し、クライアントの次のアクション（プレビュー確認 → 公開 GO 判断）まで明示している。
