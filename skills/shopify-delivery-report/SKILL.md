# Shopify Delivery Report

Shopify テーマ開発の納品・進捗報告文を半自動生成する。素材収集 → 宛先形式の確認 → テンプレートに沿った組み立て → クリップボードへコピー → （任意）実績記録、の順で進める。

報告文のテンプレートと文例は [references/report-templates.md](references/report-templates.md) を参照。

## Step 1: 素材収集

このセッション（またはユーザーが指定した範囲）の実装内容を集める。

| 素材 | 取得方法 |
|---|---|
| コミット履歴 | `git log --oneline`（範囲指定例: `git log --oneline main..HEAD` / `git log --oneline --since="today 00:00"`） |
| 変更ファイル | `git diff --stat`（同上の範囲で。例: `git diff --stat main...HEAD`） |
| 検証結果 | shopify-verifier のレポートがあればその内容（何を検証し、何が合格したか） |
| 対応した要件 | セッション内でユーザーから受けた依頼・修正指示 |

- git 情報が取れない場合（未コミット・リポ外）は、会話内容から実装項目を列挙してユーザーに確認する
- 収集した素材は報告文の「原材料」。**このままの表現で報告文に貼らない**（Step 3 で機能単位へ翻訳する）

## Step 2: 宛先形式の確認

ユーザーに 1 回だけ確認する（**デフォルト: Slack**）:

- **Slack 向け** — 簡潔・箇条書き・カジュアル敬語。日常の進捗共有・対応完了連絡向き
- **メール向け** — 丁寧・挨拶文と締めの文付き。納品報告・節目の報告向き

あわせて、案件名や宛先（メールなら会社名・担当者名）が会話から分からなければここで聞く。

## Step 3: 報告文の構成

[references/report-templates.md](references/report-templates.md) のテンプレートに従い、以下の 5 部構成で組み立てる:

1. **対応内容サマリ**（1〜3 行）— 今回何をやったかの要約
2. **変更点の一覧** — ファイル名ではなく**クライアントに意味が通じる機能単位**で書く（対訳例は references 参照）
3. **検証済み事項** — 何をどう確認したか（例: PC / スマホ両方で表示確認、既存ページへの影響なし）
4. **クライアントに確認してほしい点** — 判断依頼・素材提供依頼・レビュー依頼など、相手のアクションが必要な事項
5. **次のアクション・保留事項** — 今後の予定、ペンディング項目

## Step 4: 安全ガード（必ず適用）

- **金額・工数は自動で書かない** — 言及が必要な箇所には `【工数: 記入】` のようなプレースホルダを置き、ユーザー自身に埋めてもらう
- **プレビュー URL を含める場合** — 「この URL はクライアントに共有してよいものか」をユーザーに確認してから本文に入れる
- **内部用語の翻訳** — `c-` プレフィックス、Layer 3、schema、snippet、metafield などの内部語彙をクライアント語彙に変換する（対訳例は references 参照）

## Step 5: 出力

1. 本文を `printf '%s' "..." | pbcopy` でクリップボードへ直接コピーする
2. 「クリップボードにコピーしました」と報告する
3. 続けてチャット欄にも全文を表示する（ユーザーが目視確認・微修正できるように）
4. pbcopy が使えない環境ではチャット表示のみとする

## Step 6: 実績記録（オプション）

報告後に「今回の作業時間を実績 DB に記録しますか？」と確認する。Yes の場合:

- 記録先: `${SHOPIFY_ASSETS_DIR:-$HOME/Developer/Waggy/shopify-assets}/track-record.csv`
- 形式: `日付,案件名,作業種別,内容サマリ,時間h`
- ファイルが無ければヘッダ行付きで新規作成する
- 作業時間はユーザーに聞く（自動推定しない）

```bash
CSV="${SHOPIFY_ASSETS_DIR:-$HOME/Developer/Waggy/shopify-assets}/track-record.csv"
mkdir -p "$(dirname "$CSV")"
[ -f "$CSV" ] || printf '%s\n' '日付,案件名,作業種別,内容サマリ,時間h' > "$CSV"
printf '%s\n' '2026-07-07,サンプル案件,セクション実装,メインビジュアル新設,2.5' >> "$CSV"
```

この実績データは、将来の見積スキルの実績根拠データとして使う。
