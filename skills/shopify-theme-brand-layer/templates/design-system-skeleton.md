## §0 このドキュメントの読み方

### 0.1 テーマの 4 レイヤー構造

| 層 | 名前 | 触り方 | 具体物 |
|---|---|---|---|
| **Layer 1** | **{テーマ名} 標準コード**（変更禁止） | 編集しない | `assets/theme.css` 等 |
| **Layer 2** | **{テーマ名} 標準設定値**（管理画面で操作可能） | 管理画面 or json 編集 | `config/settings_data.json` の既存項目 |
| **Layer 3** | **Brand 層**（ストア固有上書き） | `brand-*.css` を追加 | `assets/brand-*.css` / `--brand-*` 変数 / `.{base}--brand-{name}` modifier |
| **Layer 4** | **新規セクション / コンポーネント** | `c-*` で新規作成 | `sections/c-*.liquid` 等 |

### 0.2 真実の所在（Layer of Truth）

| 真実 | 場所 | 役割 |
|---------|-----|------|
| **デザイン真実** | [Figma]({FIGMA_URL}) | 見た目（色・余白・タイポ）の最終決定権 |
| **設定真実 / 値** | `config/settings_data.json` + Shopify 管理画面 | テーマグローバル設定（Layer 2） |
| **{テーマ名} 実装** | `assets/theme.css`（Layer 1） | 変更禁止。クラス定義の出発点 |
| **Brand 実装** | `assets/brand-*.css`（Layer 3） | {テーマ名} 上書き + ストア固有トークン定義 |
| **新規コンポーネント** | `assets/c-*.css` / `sections/c-*.liquid`（Layer 4） | 単発で追加するパーツ |
| **対応表（本書）** | `document/design-system.md` | Figma 名 → Layer 1〜4 のどこにマップされるかの一覧 |

### 0.3 実装ステータス凡例

| マーカー | 意味 |
|---------|------|
| ✅ **テーマデフォルト一致** | Figma 仕様 = 現状の {テーマ名} デフォルト。何もしなくていい |
| ⚠️ **設定値要調整** | Layer 2 の値を書き換えれば Figma に一致 |
| 🛠 **Brand 層で上書き必要** | Layer 3 の `brand-*.css` で既存クラスの値を上書き |
| 🆕 **新規実装必要** | 該当する既存クラスがない。Layer 4 の `c-*` を新規作成、または Layer 3 で新規 modifier を追加 |

## §9 AI 開発フロー

### 9.1 Figma URL を渡された場合の標準手順

1. 本ドキュメントを最初に読む
2. Figma MCP で `get_design_context` を呼ぶ
3. 取得したコンポーネントを §3〜§6 のカタログと突き合わせ
4. マッピング判断と層の選択（§9.2）
5. 値の出所を §7 / §8 に追記

### 9.2 Figma → コードのマッピング判断フロー

1. **Layer 2 で表現可能か？** → 設定値の調整だけ
2. **既存クラスの値を変えれば足りるか？** → Layer 3 値上書き
3. **既存クラスに新しいバリエーションを足したいか？** → Layer 3 modifier
4. **完全新規セクションか？** → Layer 4

### 9.3 Layer 3 (Brand 層) 追加時のチェックリスト

- [ ] Figma 名から命名（`Btn / Outline / White` → `.button--brand-outline-white`）
- [ ] 既存クラスで代替可能か検証
- [ ] 値はトークン経由で参照（ハードコード禁止）
- [ ] 本書 §3〜§8 に追記

### 9.4 Layer 4 (新規セクション) 追加時のチェックリスト

- [ ] `sections/c-{name}.liquid` 等の規約遵守
- [ ] 内部で Layer 3 トークンを活用

### 9.5 Figma 変更時の更新フロー

1. Figma 側で変更
2. `get_design_context` で差分把握
3. トークン値だけの変更 → §7 を更新
4. 構造変更 → 該当 `brand-*.css` または `c-*.liquid` を更新
5. PR 作成時、本書「最終更新」と Figma node ID を更新

### 9.6 実装時の落とし穴

- セクション individual `<style>` を尊重する（slideshow 等）
- `settings_data.json` の未定義キーは新規追加
- {テーマ名} の責務分離を尊重する（縦は line-height、横は padding 等）
- 検証は同値クラスで（対象 1 ケース + 対象外 1 ケース）
