# 汎用化・機密スクラブ チェックリスト

harvest 時（Step 3）に全項目を確認する。1 つでも ❌ のまま書き込んではならない。

## A. 機密・識別情報の除去（必須・妥協なし）

- [ ] ストアドメイン（`*.myshopify.com` / 独自ドメイン）を含まない
- [ ] プレビュー URL・テーマ ID・セクション ID の実値を含まない
- [ ] API キー・アクセストークン・Webhook URL を含まない
- [ ] ピクセル ID・計測タグの実アカウント ID（adebis の pid 等）を含まない → `{{PLACEHOLDER}}` 化
- [ ] 顧客・注文・未公開商品/価格のデータを含まない
- [ ] クライアントの未公開情報（キャンペーン内容・リリース日等）を含まない

## B. テーマ依存の切り離し

- [ ] 翻訳キー（`{{ 't:...' | t }}` / `t:sections.*`）→ 生テキスト + 「翻訳キーに戻す」旨を汎用化メモへ
- [ ] テーマ固有のカスタム要素（`<scroll-carousel>` 等）・JS グローバルへの依存を汎用化メモに明記
- [ ] `color_scheme` / `settings.*` への参照は「テーマ側で用意が必要な設定」として明記
- [ ] Focal/Dawn 等どのテーマで動作確認したかを frontmatter `themes:` に記録
- [ ] テーマから切り離せない構造（section 専用 wrapper 等）は type を `section` / `schema-pattern` にして構造ごと保存

## C. 値の変数化

- [ ] 色の raw value（`#fff` 等）→ `var(--token-name, #fff)` 形式（フォールバック付き）
- [ ] サイズ・余白の raw value → 変数化 or 「差し替えポイント」コメント
- [ ] フォント指定 → テーマの font 設定参照に置換 or 明記
- [ ] 画像 URL・Figma 由来のアセット参照 → プレースホルダ化

## D. 権利面の判断

- [ ] 受託契約で成果物の権利がクライアント帰属の場合、コードの丸ごと複製ではなく**構造・技法・パターン**として抽象化した
- [ ] デザインそのもの（見た目の完全再現データ）ではなく実装パターンとして保存した
- [ ] 判断に迷う場合はユーザーに確認した

## E. 再利用性の担保

- [ ] カードの「使い方」だけ読めば流用できる（元案件のコンテキスト不要）
- [ ] 引数・依存・前提（Liquid オブジェクト、親要素の構造）を明記した
- [ ] 「汎用化メモ」に流用時の必須差し替え箇所を列挙した
